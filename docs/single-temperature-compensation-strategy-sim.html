目标与口径

  • 目标 1不同温度下，同一距离的外光路幅值 AX 尽量一致。
  • 目标 2不同温度下，同一距离的 4 频相位补偿后尽量一致。
  • 统一口径先做 工作点补偿，再做 相位补偿，两层逻辑分开，不混成单一 coeff。
  • 单发专用单发以外光路为主，运行时先参考 VBR 粗定位，再用 AX 小闭环修正工作点。

核心原则

为什么不能只看 coeff

旧数据已经证明：同样的 coeff = APD / VBR，不同温度下 AX 仍会明显变化，所以 coeff 只能做辅助相对量，不能单独代表工作点。

为什么先补工作点

如果不同温度下增益状态都不一致，后面的相位补偿表会被“工作点漂移”污染，所以必须先把 AX 拉回同一区间，再做相位表补偿。

单发最终建议口径：本机常温基准 + 当前 VBR 相对变化 + 运行时 AX 小闭环 + 当前工作点相位查表。

工作点模型（先让幅值一致）

目标工作区先固定为：AX_target = 18000。从每个温度点内部，反推出最接近目标幅值的工作 APD。

测距时怎么执行（运行时闭环）

步骤输入动作输出
1. 读当前状态VBR_now先获取当前温度状态代表量VBR_now
2. 求 APD 初值VBR_now、工作点模型按模型求 APD_initAPD_init
3. 幅值小闭环APD_init、AX_target打一枪看外光路 AX，不够就加 APD，过高就减 APDAPD_work_now、AX_now
4. 采 4 频相位APD_work_now采 P129/P134/P159/P160原始 4 频相位
5. 查相位表freq + VBR_now + AX_now命中二维补偿表phase_corr[4]
6. 组合测距补偿后相位进入多频组合dist、quality

运行时模拟例子

已知： VBR_now = 930 AX_target = 18000 步骤： 1) APD_init = 810 + 1.07 × (930 - 900) = 842 2) 实测 AX_now = 17600，略低 3) APD += 1，得到 APD_work_now = 843 4) 再测 AX_now = 18100，已进入目标区 5) 采 4 频 raw phase 6) 按 (freq, VBR=930, AX≈18100) 查表补偿 7) 用 final phase 做多频组合测距

建议公式（统一平台单发版）

校准阶段保存： temp_ref vbr_ref apd_work_ref ax_ref phase_ref_table workpoint_comp_params phase_corr_table[freq][VBR_bin][AX_bin] 运行时执行： 1) vbr_now = read_vbr() 2) apd_init = model_workpoint(vbr_now) 3) ax_now = quick_sample_outer(apd_init) 4) while ax_now 不在 target window: apd_init += step(ax_target - ax_now) ax_now = quick_sample_outer(apd_init) 5) raw_phase[4] = capture_phase(apd_init) 6) corr[4] = lookup_phase_corr(freq, vbr_now, ax_now) 7) final_phase[4] = raw_phase[4] + corr[4] 8) dist = combine_multi_freq(final_phase)
建议保留原则：工作点控制以相对模型为主，绝对阈值只做保护。也就是 VBR 相对变化 + AX 小闭环 主导，APD_min/max、AX_min/max、饱和阈值 只负责防跑飞。

结论

  • 原策略方向是对的：先温扫建模，再工作点补偿，再同工作点相位补偿。
  • 需要明确统一平台口径：不能再把单发温补简化成单一 coeff 逻辑。
  • 单发最稳的执行链应固定为：VBR -> APD 初值 -> AX 小闭环 -> 相位查表 -> 组合测距。
  • 本页是模拟数据版，后续可继续追加“实机温扫版”和“固件映射版”。
